AI MEMORY SYSTEM / SUBMISSION PROTOTYPE

不是保存事情，
而是留住事情之后的余波。

这是一个私人记忆系统：当你还说不清楚、但不想弄丢某种感觉时，先留下几个词；未来再由 Back 轻轻把你带回去。

“趁余波还在，先留下几个词。
未来的你，再回来认领。”

这次提交最想验证什么

不是“AI 能不能把日记写得更漂亮”，而是：人在很累、很空、还没想清楚的时候，能否不整理、不分类，只留下原话；而“之后再说”能否在未来成为一次回望，而不是一个待办。

三个亮点

01 · 原话优先

系统不把用户的感受压缩成标签或总结。原话始终在卡片上，解释权留给未来的用户自己。

02 · “之后再说”是有效状态

暂放不是取消，也不是未完成任务。它让感受先被保存，等未来有力气时再回来命名。

03 · Back 不是搜索

它不问“你要找什么”，而是把一张曾经的卡递回来，让人重新碰到当时的 Moment、World、Presence 或 Self。

请这样体验（约 2 分钟）

  1. 首页已经预置一段“卡在 API 上的下午”。先读它，点击「留下」。
  2. 看到回执后，不编辑分类，点击「先放着（继续体验）」。
  3. 进入 Back，点击「收件」，再点击「回去看看」。
  4. 选择「回到一个曾经的我」。
  5. 点击「补一句」，写下你现在回看这段话时想补的内容，再点击「提交」。
开始体验 →

这版还没有做到什么

这是一个受控的概念验证，不是完整产品。当前体验使用预置的高质量样本和本地 mock 规则；真实 LLM 接入仍停留在独立训练台，尚未进入这条产品主流程。卡片仅保存于当前浏览器，刷新/换设备不会形成真实的长期个人记忆库。

图片、语音、搜索/管理、真实多用户测试、完整的自动回访机制也都还未完成。当前最重要的不是功能齐全，而是验证：这条“留下 → 暂放 → 回去 → 补一句”的路径，是否真的让人回去一点。

接下来

先收集对这条核心路径的评价；再接入真实模型和持久化，让用户自己的低能量输入也能被安全地接住、延后确认，并在未来回到他们手里。

请把这里当作一段可体验的产品片段，而非完整产品承诺。
